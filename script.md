## [슬라이드 01 — 커버] `약 40초`

안녕하세요, 발표를 맡게 된 김지현입니다.

저희 팀은 포항자동화사업실 인텔리전스조업그룹의 PIXEL 팀으로, 김지현, 허도원, 심경보 세 명으로 구성되어 있습니다.

PIXEL은 **POSCO Intelligent eXperience Layout**의 약자로, 한 문장으로 표현하면 —
**자연어로 말하면 AI가 화면을 이해하고 그려주는 플랫폼**입니다.

## [슬라이드 02 — Problem] `약 1분`

지금 현장에서 단 한 개의 화면 요건을 확정하기까지 **최소 11시간**이 걸립니다.
왜 이렇게 걸릴까요? 네 가지 구조적 문제가 맞물려 있습니다.

첫째, 설계자가 PPT로 화면을 직접 그리며 반복 수정에 시간을 쓰는 **작업 지연**.
둘째, 현업과 IT 사이의 도메인 언어 차이로 요구사항이 계속 어긋나는 **소통 병목**.
셋째, 메일·Teams·통화에 흩어진 대화로 최신 내용을 추적할 수 없는 **분산된 이력**.
넷째, 과거 설계 자산이 남지 않아 비슷한 화면도 매번 처음부터 다시 만드는 **자산화 실패**.

이 네 가지가 서로 맞물려 악순환을 만들고 있습니다.

기존 방식은 두 가지입니다.

## [슬라이드 08 — PIXEL 파이프라인] `약 1분 30초`

PIXEL은 다르게 했습니다. **말하면, AI가 이해하고 그려줍니다.**

파이프라인은 5단계입니다.

**1단계, 요건 입력** — 현업이 자연어, Excel, 이미지 어떤 형태로든 화면 요건을 입력합니다.

**2단계, 온톨로지 추출** — AI가 입력된 요건을 MES·SCM·ERP 도메인 지식 체계로 구조화하고, 어떤 부분이 빠져 있는지 파악합니다. Claude Sonnet이 담당합니다.

**3단계, Q&A 루프** — 파악된 모호하거나 누락된 부분에 대해 AI가 추가 질문을 자동 생성합니다. 최대 5회, LangGraph 기반으로 동작합니다.

**4단계, 목업 생성** — 확정된 요건을 React 코드로 변환하고, SSE 스트리밍으로 화면이 눈앞에서 실시간으로 완성됩니다.

**5단계, 리뷰 & 자산화** — 검토·승인 워크플로우를 거쳐 설계 자산이 GraphRAG에 쌓입니다.

## [슬라이드 05 — 온톨로지] `약 55초`

2단계에서 말씀드린 온톨로지, 잠깐 짚고 넘어가겠습니다.

**엔티티가 "무엇이 있는가"를 정의한다면, 온톨로지는 "그것들이 업무 맥락에서 어떤 의미로 서로 연결되는가"까지 정의한 도메인 지식 지도입니다.**

예를 들어, AI가 "생산실적 화면"이라는 말을 받았을 때 — DB 스키마만 알면 단순 컬럼 나열 화면을 만들지만, 온톨로지가 있으면 설비코드 필터, 조업조건 연결, 차지번호 추적, 시간대별 집계 차트까지 맥락에 맞게 판단해서 만들 수 있습니다.

온톨로지는 4가지 역할을 합니다. AI Hallucination 방지, 정확한 Q&A 질문 생성 기준, 화면 생성 품질 일관성, 그리고 GraphRAG 기반 유사 설계 재사용.

## [슬라이드 06 — Demo] `약 1분 20초`

이제 실제로 작동하는 걸 보여드리겠습니다. 주요 기능은 다섯 가지입니다.

**1번, 화면 UI 생성** — 자연어를 입력하면 AI가 목업을 자동으로 만들어냅니다. 기존 설계 결과물 대비 82% 유사도입니다.

**2번, AI 수정** — 생성된 목업에 "이 버튼 오른쪽으로 옮겨줘"라고 말하면 AI가 즉시 반영합니다.

**3번, 버전 비교** — 이전 버전과 현재 버전을 나란히 비교할 수 있습니다.

**4번, 리뷰 워크플로우** — 생성부터 검토의뢰, 승인요청, 최종승인까지 단일 플랫폼에서 처리됩니다.

**5번, PPT Export** — Gamma API를 통해 목업과 스펙이 화면설계서 PPT로 자동 변환됩니다. 문서 작성에 쓰던 시간이 0이 됩니다.

## [슬라이드 09 — Impact] `약 55초`

실제 결과물 기준으로 비교해봤습니다.

화면 초안 작성, **10분 이내**. 초안 하나에 하루를 쓰던 흐름이 대화 몇 마디로 끝납니다.

검토·승인 왕복 시간, **50% 단축**. 피드백과 이력이 한 플랫폼에 묶이기 때문입니다.

화면설계서 파일 작성 시간, **0시간**. 자동 생성되기 때문에 별도 작성이 없습니다.

생성 품질, **유사도 90%**. 처음부터 수정 부담이 낮은 초안이 나옵니다.

## [슬라이드 10 — Roadmap] `약 30초`

지금 단계인 Phase 1은 MES·SCM·ERP 3개 도메인, 자연어 목업 생성 98% 성공률, 검토·승인 워크플로우까지 완성된 상태입니다.

Phase 2에서는 GraphRAG 유사 설계 조회를 실제 목업 생성 프롬프트에 반영하고 실시간 협업을 추가합니다.

Phase 3은 전사 표준 플랫폼으로 확산하고, 누적 설계 자산이 AI 품질을 스스로 높이는 선순환 구조가 목표입니다.

## [슬라이드 11 — Journey 챌린지] `약 40초`

5주가 쉽지 않았습니다.
1주차엔 Supabase 전면 도입을 결정했고, 2주차엔 LangGraph 적용 범위를 Q&A 핵심 흐름으로 좁히고 전체 오케스트레이션은 FastAPI로 가는 트레이드오프를 선택했습니다. AI 비용 최적화를 위해 품질이 중요한 온톨로지·목업 생성은 Sonnet, Q&A는 Haiku로 역할을 분리했고, 3주차엔 DB 스키마를 개발 중반에 전면 재설계했습니다. 마지막으로 GraphRAG 조회 참조는 Phase 2로 우선순위를 조정하는 판단을 내렸습니다.

이 결정들 하나하나가 지금의 PIXEL을 만들었습니다.
